# Design Doc: 市场温度计 (Market Monitor)

**状态**: Approved  
**日期**: 2026-04-14  
**作者**: Haolun Yan

---

## 1. 目标

在 FamilyFund Dashboard 新增 **Tab 5「市场温度计」**，整合以下市场信号，辅助大类资产定投倍数决策：

- 5个标的的 MA60 / MA200 乖离率（A股用MA60为主，美股/黄金用MA200为主）
- VIX 恐慌指数
- 标普500 / 纳指100 PE 估值
- 基于 PE × VIX 矩阵输出每个标的的**定投倍数建议**（如 0.5x / 1.0x / 顶格 / 暂停）

系统只输出倍数，用户以自身基准定投金额执行，无需告知系统基准金额。

## 3. 乖离率参考规则

### 3.1 主要参考均线

| 标的 | 主要参考 | 次要参考 | 原因 |
|------|----------|----------|------|
| CSI 300 | MA60 | MA200 | A股短周期性强，MA60 更敏感 |
| 中证A500 | MA60 | MA200 | 同上 |
| 黄金 | MA200 | MA60 | 宏观对冲资产，长周期驱动，MA60 噪音大 |
| 纳指100 | MA200 | MA60 | 机构普遍看 MA200，长期趋势信号更可靠 |
| 标普500 | MA200 | MA60 | 同上 |

UI 展示时，主要参考列视觉权重更高（加粗或放大）。

### 3.2 乖离率信号分档

乖离率 = `(当前价 − MAn) / MAn × 100%`

| 区间 | 信号 | 颜色 |
|------|------|------|
| ≤ −10% | 深度超卖 | 🔵 蓝 |
| −10% ~ −5% | 超卖 | 🟢 绿 |
| −5% ~ +8% | 正常 | ⚪ 灰 |
| +8% ~ +15% | 偏高 | 🟡 黄 |
| > +15% | 超买 | 🔴 红 |

乖离率本身只做展示和颜色提示，**不单独触发定投倍数**。倍数由下方 PE × VIX 矩阵决定（仅适用于标普/纳指）。A股和黄金暂无矩阵，只用乖离率颜色提示。

## 4. 定投倍数矩阵（PE × VIX）

仅适用于**标普500**和**纳指100**，来源于用户确认的参考框架。

### 4.1 标普500 定投倍数矩阵

| PE \ VIX | < 18 | 18 – 25 | 25 – 35 | > 35 |
|----------|------|---------|---------|------|
| > 32     | 暂停 | 暂停    | 观望    | 0.3x |
| 29 – 32  | 暂停 | 0.3x    | 0.5x    | 0.8x |
| 26 – 29  | 0.3x | 0.5x    | 0.8x    | 1.2x |
| 23 – 26  | 0.6x | 0.8x    | 1.5x    | 2.0x |
| 20 – 23  | 1.2x | 2.0x    | 3.0x    | 4.5x |
| 17 – 20  | 2.5x | 4.0x    | 7.0x    | 10.0x |
| 14 – 17  | 5.0x | 8.0x    | 14.0x   | 顶格 |
| < 14     | 顶格 | 顶格    | 顶格    | 顶格 |

### 4.2 纳指100 定投倍数矩阵

| PE \ VIX | < 18 | 18 – 24 | 24 – 31 | > 31 |
|----------|------|---------|---------|------|
| > 37     | 暂停 | 暂停    | 观望    | 0.3x |
| 35 – 37  | 暂停 | 0.2x    | 0.4x    | 0.6x |
| 32 – 35  | 0.3x | 0.5x    | 0.8x    | 1.2x |
| 28 – 32  | 0.6x | 0.8x    | 1.2x    | 2.0x |
| 24 – 28  | 1.0x | 1.5x    | 2.5x    | 4.0x |
| 20 – 24  | 2.0x | 3.5x    | 6.0x    | 9.0x |
| 16 – 20  | 4.0x | 7.0x    | 12.0x   | 顶格 |
| < 16     | 顶格 | 顶格    | 顶格    | 顶格 |

**"暂停"**：当前估值过高且市场平静，不建议定投。  
**"观望"**：信号矛盾，维持最小仓位或不操作。  
**"顶格"**：极度低估+极度恐慌，全力加仓。

---

## 5. 无综合信号设计

不做跨标的的综合投票信号。原因：用户对每个标的有独立持仓目标，标的间信号不应互相抵消（黄金超卖不会因标普超买而被中和）。每个标的独立展示乖离率信号和倍数建议，用户逐标的决策。

---

## 6. 数据缓存策略

独立缓存文件（不与 benchmark_cache 合并）：

```
$FAMILYFUND_DATA/market_cache.json
```

```json
{
  "csi300":    {"price": 3880.1, "ma60": 3820.0, "ma200": 3600.0, "updated": "2026-04-14"},
  "csi_a500":  {"price": 5792.8, "ma60": 5710.0, "ma200": 5500.0, "updated": "2026-04-14"},
  "gold":      {"price": 4808.3, "ma60": 4650.0, "ma200": 4200.0, "updated": "2026-04-14"},
  "ndx100":    {"price": 19200.0,"ma60": 19800.0,"ma200": 20100.0,"updated": "2026-04-14"},
  "sp500":     {"price": 5200.0, "ma60": 5400.0, "ma200": 5500.0, "updated": "2026-04-14"},
  "vix":       {"price": 18.55,  "updated": "2026-04-14"},
  "pe_sp500":  {"value": 27.24,  "source": "VOO trailingPE", "manual_override": null, "updated": "2026-04-14"},
  "pe_ndx100": {"value": 32.62,  "source": "QQQ trailingPE", "manual_override": null, "updated": "2026-04-14"}
}
```

规则：
- 当天已更新 → 直接读缓存，不发网络请求
- 过期 → 尝试拉新；失败时 fallback 到旧缓存，UI 标注"数据陈旧 (YYYY-MM-DD)"
- `manual_override` 非 null 时，PE 使用手动值，忽略自动值（用于网络不可达时）
- 各标的独立更新，某个接口失败不影响其他

## 8. `src/market_monitor.py` 模块设计

### 公开 API

```python
get_market_data() -> dict
# 主入口。拉取/读缓存，返回所有标的的 price/ma60/ma200/vix/pe，带 meta 信息

compute_bias(entry: dict) -> dict
# 输入单个标的的 {price, ma60, ma200}
# 返回 {bias60: float, bias200: float, signal60: str, signal200: str}

lookup_multiplier(pe: float, vix: float, target: str) -> str
# 查矩阵，返回倍数字符串："暂停" / "观望" / "0.3x" / ... / "顶格"
# target: "sp500" 或 "ndx100"
```

### 内部结构

```
market_monitor.py
├── TARGETS             # 标的配置：名称/数据源/ticker/主要均线
├── BIAS_THRESHOLDS     # 乖离率分档常量
├── SP500_MATRIX        # 标普 PE×VIX 二维查找表
├── NDX100_MATRIX       # 纳指 PE×VIX 二维查找表
├── _load_cache()
├── _save_cache()
├── _cache_is_fresh()
├── _fetch_akshare(symbol)     # CSI300, 中证A500，返回 pd.Series(close, index=date)
├── _fetch_yfinance(symbol)    # 金价/纳指/标普/VIX
├── _fetch_pe(symbol)          # VOO / QQQ trailingPE via yfinance info
├── _compute_ma(series, window) -> float   # 最新 MA 值
├── get_market_data()          # 带缓存的主流程
├── compute_bias(entry)
└── lookup_multiplier(pe, vix, target)
```

### 矩阵查找实现思路

```python

    ["0.3x", "0.5x", "0.8x", "1.2x"], # 26-29
    ["0.6x", "0.8x", "1.5x", "2.0x"], # 23-26
    ["1.2x", "2.0x", "3.0x", "4.5x"], # 20-23
    ["2.5x", "4.0x", "7.0x","10.0x"], # 17-20
    ["5.0x", "8.0x","14.0x", "顶格"], # 14-17

```

## 10. 历史走势图

**第一版不实现。** 只展示当前值 + 信号颜色。

理由：当前值已足够触发调仓决策；历史图是复盘工具，不在 weekly update workflow 关键路径上。后续如有需要再扩展。

## 11. 已确认设计决策汇总

| 决策点 | 结论 |
|--------|------|
| 中证A500数据源 | akshare `sh000510` |
| PE ticker | VOO（标普）/ QQQ（纳指）|
| 主要均线 | A股→MA60，美股/黄金→MA200 |
| 乖离率正常区间上限 | +8% |
| 定投倍数矩阵 | 直接采用用户参考框架（见§4）|
| 综合信号 | 不做，各标的独立展示 |
| 黄金单位 | USD/oz |
| 历史走势图 | 第一版不做 |
| 基准定投金额 | 系统不需要，只输出倍数 |
