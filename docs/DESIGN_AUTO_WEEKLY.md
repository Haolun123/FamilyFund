> **状态**：待实现（P2）
> **日期**：2026-05-09（更新）

---

## 功能定位

减少每周对账的手动操作量。当前最大的摩擦点：

1. **查价格**：需要打开各平台查最新净值/价格，逐行填入 `Current_Price`
2. **记调仓**：买入确认后需要手动填写份额、净值、NCF

目标：把每周对账从"逐行手填"变成"确认+修改少数异常"。

### 天天基金接口

6位纯数字 Code 的国内公募基金，走天天基金净值接口：

```
GET https://api.fund.eastmoney.com/f10/lsjz?fundCode=018738&pageIndex=1&pageSize=1
Header: Referer: https://fund.eastmoney.com/
```

返回最新净值（`DWJZ` 字段）和净值日期（`JZRQ`）。

### Code 格式路由规则

```python
def _route_code(code: str) -> str:
    if re.match(r'^\d{6}$', code):           return 'eastmoney'   # 国内公募基金
    if code.endswith('.SS') or code.endswith('.SZ'): return 'akshare'   # A股个股
    if code.startswith('HK'):                return 'yfinance_hk' # 港股
    if code in ('GOLD', 'GOLD.P'):           return 'gold'        # 黄金
    if code == 'SAP.DE':                     return 'yfinance_us' # 美股/ADR
    if code == 'CASH':                       return 'fixed_1'     # 现金
    return 'manual'                                                 # 无法自动
```

## 功能二：短信解析自动录入

### 架构决策：并入调仓辅助器

短信解析**不直接写入 snapshot**，而是作为调仓辅助器的"自动填写"入口：

```
粘贴短信 → 解析 → 自动填入调仓辅助器对应字段
         → 用户确认 → 调仓辅助器正常走保存流程
```

**原因：**
- 调仓辅助器有 Cash 余额扣减、NCF 计算等校验逻辑，直接 append 会绕过
- 保持"调仓辅助器是唯一写入入口"的原则，系统行为可预期
- 短信解析的价值是省去手动查净值和份额，确认动作仍由用户完成

### 实际短信样本分析（reference/message.md）

共4种格式，覆盖主要定投场景：

#### 格式A：博时基金
```
【博时基金】尊敬的严浩伦，您于2026年05月06日通过博时直销申购
博时标普500ETF联接E 240元05月08日确认成功，份额为44.19份，净值为5.4308。
```
提取：申购日(05-06) → 确认日(05-08)、基金名、金额(240)、份额(44.19)、净值(5.4308)

#### 格式B：南方基金（定投格式）
```
【南方基金】尊敬的严浩伦先生，您5月7日定投南方中证A500ETF联接A基金1,050.00元
于5月8日确认成功，确认份额797.75份，成交净值1.3162。
```
提取：定投日(5-7) → 确认日(5-8)、基金名、金额(1050.00，含逗号)、份额(797.75)、净值(1.3162)

注：南方基金短信中的定投日期格式为 `X月X日`（不含年份），确认日同格式。

#### 格式C：南方基金（QDII格式，与格式B相同）
同格式B，用同一套正则覆盖。

#### 格式D：招商银行黄金积存金
```
【招商银行】尊敬的客户，您的XXXXXXXX号黄金智能定投计划已于2026年04月30日
扣款成功，为您成功定投1.7000克黄金份额，扣款金额人民币1727.12元，扣款账户尾号8097
```
完全不同的格式，提取：日期(2026-04-30)、克数(1.7)、金额(1727.12)
**不提取净值**（克数即份额，净值=金额/克数自动算）

### 正则设计

```python
# 格式A：博时、多数基金公司（"申购"关键词）
PATTERN_SUBSCRIPTION = re.compile(
    r'于(\d{4}年\d{2}月\d{2}日)'          # 申购日期
    r'.*?(申购|买入)'
    r'(.+?)\s+'                            # 基金名称
    r'([\d,]+\.?\d*)元'                    # 金额（含逗号）
    r'.*?(\d{1,2}月\d{1,2}日)确认成功'    # 确认日期
    r'.*?份额为([\d.]+)份'                  # 份额
    r'.*?净值为([\d.]+)',                   # 净值
    re.DOTALL
)

# 格式B/C：南方基金定投格式（"定投"关键词）
PATTERN_DCA = re.compile(
    r'您(\d{1,2}月\d{1,2}日)定投'         # 定投日期
    r'(.+?)基金'                           # 基金名称
    r'([\d,]+\.?\d*)元'                    # 金额
    r'于(\d{1,2}月\d{1,2}日)确认成功'     # 确认日期
    r'.*?确认份额([\d.]+)份'               # 份额
    r'.*?成交净值([\d.]+)',                 # 净值
    re.DOTALL
)

# 格式D：招商银行黄金积存金
PATTERN_GOLD = re.compile(
    r'已于(\d{4}年\d{2}月\d{2}日)扣款成功'  # 日期
    r'.*?定投([\d.]+)克黄金'                  # 克数
    r'.*?扣款金额人民币([\d.]+)元',            # 金额
    re.DOTALL
)
```

### 日期处理

- 格式A：含年份，直接解析
- 格式B/C：只有 `X月X日`，补当前年份（若月份 > 当前月，取上一年）
- 格式D：含年份，直接解析

### 基金名称匹配

短信里的名称（如"博时标普500ETF联接E"）需要匹配到 portfolio.csv 的 `Name` 字段（如"博时标普500 E类"）：

```python
# 1. 关键词包含匹配（优先）：提取数字+关键词
#    "博时标普500ETF联接E" → 关键词["博时","标普","500","E"] 
#    匹配 "博时标普500 E类" ✓
# 2. difflib.get_close_matches 模糊匹配（备用）
# 3. 无法匹配：下拉选择（展示所有持仓供用户手选）
```

### 黄金特殊处理

黄金短信不含净值，需要反算：
- `Shares` 变化：+1.7（克）
- `Current_Price`：`金额 / 克数` = `1727.12 / 1.7` = `1016.01`（元/克）
- `NCF`：+1727.12

### 写入逻辑

解析结果写入当周快照时：
- **新增定投**（短信确认日 = 当周快照日期）：在当周快照中找到对应持仓行，更新：
  - `Shares` += 确认份额
  - `Current_Price` = 确认净值
  - `NCF` = +申购金额
- **Current_Price 冲突**：若同一持仓当周有多条短信，取最新净值日期的净值

### 交互流程

```
[Ledger Tab → 调仓辅助器区域] 新增「📱 从短信导入」入口（折叠区）
  → 文本框：粘贴短信（支持多条，用空行分隔）
  → 点击「解析」
  → 展示解析预览表（可编辑）：

    2026-05-08 | 买入 | 博时标普500 E类    | 240    | 44.19 | 5.4308
    2026-05-08 | 买入 | 南方中证A500 A类   | 1050   | 797.75 | 1.3162
    2026-04-30 | 买入 | 黄金（积存金）     | 1727   | 1.7g  | 1016.01
  → 无法匹配的行：下拉手选持仓
  → 点击「填入调仓辅助器」
  → 对应字段自动填入调仓辅助器（买入金额、净值、份额）

```

## 不做 LLM 兜底的原因（修订原设计）

原设计有 LLM 兜底方案，但基于实际短信样本分析后，决定**不做**：

1. 4种格式正则已能覆盖全部实际定投短信
2. LLM 调用引入网络依赖和延迟，而短信解析是离线操作
3. 正则解析结果可预期，LLM 结果可能有幻觉风险（把金额/份额搞混）
4. 无法解析的格式用"下拉手选"兜底，成本更低

## 设计决策

- **短信解析并入调仓辅助器**：短信解析是"自动填写调仓辅助器"，不是独立写入路径。调仓辅助器是唯一数据写入入口，保持不变
- **不做全自动定时刷新**：Weekly Update 是 CIO 主动行为，需要人工确认
- **不做 LLM 兜底**：正则已覆盖全部实际样本，下拉手选兜底足够
- **短信不持久化**：短信内容仅用于当次解析，不存储
- **刷新净值和短信解析独立**：两个功能互不依赖，可分别触发
- **固定收益保留上期价格**：周周宝/月月宝/招银招睿无公开接口，显示警告提示手动确认
