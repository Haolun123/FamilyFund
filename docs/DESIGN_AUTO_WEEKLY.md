> **状态**：待实现（P2）
> **日期**：2026-05-09

---

## 功能定位

减少每周对账的手动操作量。当前最大的摩擦点：

1. **查价格**：需要打开各平台查最新净值/价格，逐行填入 `Current_Price`
2. **记调仓**：买入确认后需要手动填写份额、净值、NCF

目标：把每周对账从"逐行手填"变成"确认+修改少数异常"。

## 功能一：一键刷新净值

### 数据源

| 资产类型 | 数据源 | 接口 |
|---------|--------|------|
| 国内公募基金（QDII/指数基金）| 天天基金 | `api.fund.eastmoney.com/f10/lsjz?fundCode=018738&pageIndex=1&pageSize=1` |
| A股个股 | akshare | `stock_zh_a_hist` 或 `fast_info` |
| 港股 | yfinance | `.HK` ticker |
| 黄金 | yfinance | `GC=F` + HKD/CNY 汇率换算 |
| SAP（ADR）| yfinance | `SAP` |
| 固定收益/银行理财 | 无自动源 | 保留上期价格，标注"需手动确认" |

### 交互流程

```
[Ledger Tab] 点击「🔄 刷新净值」按钮
  → 系统自动拉取所有可自动获取标的的最新价格
  → 填入编辑区（Current_Price 列）
  → 无法自动获取的标的（固定收益等）保留上期价格，高亮提示"需确认"
  → 用户核对、修改少数异常项
  → 正常走保存流程
```

### 代码映射

`portfolio.csv` 中的 `Code` 字段已区分资产类型：
- 6位数字（如 `018738`）→ 国内公募基金 → 天天基金接口
- `601838.SS` / `0700.HK` 等 → 股票 → yfinance（复用 `yf_symbols.json` 映射）
- `GOLD` → 黄金 → yfinance `GC=F` + 汇率

新增函数 `fetch_latest_prices(raw_df) -> dict[code, price]`，统一封装各数据源。

## 功能二：短信解析自动录入调仓

### 背景

基金公司确认短信格式固定，包含所有需要录入的字段：

```
【博时基金】尊敬的严浩伦，您于2026年05月06日通过博时直销申购
博时标普500ETF联接E 240元05月08日确认成功，份额为44.19份，
净值为5.4308。
```

可提取：
- **操作类型**：申购（买入）/ 赎回（卖出）
- **基金名称**：博时标普500ETF联接E → 匹配持仓 Code
- **申购金额**：240元 → NCF
- **确认日期**：2026年05月08日 → Date
- **确认份额**：44.19份 → Shares 变化量
- **确认净值**：5.4308 → Current_Price

### 解析方案

**方案A：正则（优先）**

国内主流基金公司短信格式相对固定，用正则可覆盖大部分情况：

```python
# 博时、南方、招商等主流格式
pattern = r'于(\d{4}年\d{2}月\d{2}日).*?(申购|赎回)(.+?)\s*(\d+\.?\d*)元.*?确认成功.*?份额为(\d+\.?\d*)份.*?净值为(\d+\.?\d*)'
```

提取字段：日期、操作类型、基金名称（模糊匹配持仓）、金额、份额、净值。

**方案B：LLM 兜底**

对正则无法解析的格式（招行积存金短信格式不同），调用 GLM-4-flash 提取结构化字段，成本极低（< ¥0.001/条）。

### 交互流程

```
[Ledger Tab] 新增「📱 短信解析」入口（折叠区）
  → 文本框粘贴短信内容（支持多条，换行分隔）
  → 点击「解析」
  → 系统提取字段，展示预览表：
    日期 | 操作 | 基金名称 | 金额(NCF) | 份额变化 | 净值
    2026-05-08 | 买入 | 博时标普500ETF联接E | +240 | +44.19 | 5.4308
  → 用户确认（或修改）
  → 写入当周快照（更新对应持仓的 Shares、Current_Price、NCF）
```

### 基金名称匹配

短信里的基金名称是简称，需要模糊匹配持仓：

```python
# portfolio.csv 中的 Name 字段 vs 短信中的基金名称
# 用 difflib.get_close_matches 或简单的关键字包含匹配
# 无法自动匹配时，提供下拉让用户手选
```

### 黄金积存金特殊处理

招行积存金的短信格式不同：
```
【招商银行】您于05月06日成功买入黄金2克，成交均价XXX元/克
```
需要单独处理：克数 → 更新 Shares，金额 → NCF。

## 设计决策

- **不做全自动定时刷新**：Weekly Update 是 CIO 主动行为，需要人工确认，不应在后台静默修改 portfolio.csv
- **正则优先于 LLM**：短信解析成本低、格式固定，LLM 仅作兜底，不作主路径
- **短信不持久化**：短信内容只用于当次解析，不存储，隐私数据不落盘
