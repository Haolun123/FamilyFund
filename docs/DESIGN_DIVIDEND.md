# DESIGN: 现金分红处理

> **状态**：短期方案 A 已执行，长期方案 C 待实现
> **日期**：2026-04-25；**更新**：2026-05-16（SAP 分红口径对齐业界标准）

---

## 涉及场景

| 标的 | 分红类型 | 当前处理方式 |
|------|---------|------------|
| 成都银行、腾讯控股 | 现金分红，直接打入证券账户 | 方案 A（Cash 入金） |
| 红利低波ETF | 现金分红 / 红利复投 | 红利复投无需处理（净值自然上升）；现金分红同方案 A |
| SAP 股票 | 股票形式再投资（Dividend Reinvestment）| ✅ NCF = 0，分红复投为内部回报，体现在 NAV 上升；不计入成本基准 |
| 场外基金（纳指/标普/A500） | 净值法（分红折算净值） | ✅ 无需处理 |

**统一原则（业界公募基金全收益标准）**：分红复投 = 持股孵化的内部回报，与红利低波 ETF 底层分红复投一致，NCF = 0，不计入成本基准，收益体现在 NAV 上升。

---

## 核心矛盾：NCF 语义冲突

当前 NCF 设计：**买入 = 正数，卖出 = 负数**

现金分红若记为正 NCF，会与买入混淆，导致 `Cost_Basis = Σ NCF` 虚高：

```
成都银行：买入成本 ¥20,880，分红 ¥1,200
错误做法（分红记 NCF +1,200）：Cost_Basis = 22,080 ← 虚高
正确做法：Cost_Basis = 买入 - 分红 = 19,680
```

---

## 三种方案

### 方案 A（当前使用）：分红记为 Cash 外部入金

分红到账时，在 Cash 行记 NCF = +分红金额，备注来源。

- ✅ 简单，整体 NAV 正确
- ❌ 单资产收益失真（分红不体现在来源持仓盈亏里）

**操作规范**：
1. 调仓辅助器「＋ 外部入金」
2. 金额填税后到账金额
3. 备注填「[持仓名] 现金分红」（如「成都银行 现金分红」）

### 方案 B：分红记为持仓行负 NCF

在来源持仓记 NCF = -分红金额，Cash Total_Value 增加但 NCF = 0。

- ✅ 单资产盈亏准确
- ⚠️ 需扩展调仓辅助器，增加「分红」操作类型

### 方案 C（长期目标）：transaction.csv 新增 Dividend 类型

在 `transaction.csv` 加一条 `Type = Dividend, Amount_CNY = -分红金额`。

- ✅ 完整分红历史，支持未来收益归因
- ✅ 不改变 NCF 语义，向后兼容
- ❌ 实现最复杂，需同时查 portfolio.csv 和 transaction.csv
