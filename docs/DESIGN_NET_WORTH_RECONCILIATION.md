# DESIGN: 季度净资产核对（Net Worth Reconciliation）

> **状态**：已实现（v3，2026-07-01）
> **位置**：`src/cashflow_engine.py::compute_net_worth_reconciliation`
> **展示**：Dashboard Quarterly Report Tab，每个季度对比的"净资产核对"section

---

## 一、设计目标

回答一个核心问题：**期初到期末的净资产变化，能不能用具体的现金流来解释？**

如果能解释（残差≈0），说明数据完整、对账无误。
如果不能（残差大），说明：
- 数据有遗漏（鲨鱼记账漏记 / cashflow_log 漏录 / balance_sheet 估值未更新）
- 或者有显著的资产估值波动（SAP 股价跌、黄金涨等），残差恰好捕捉这部分

---

## 二、核心公式

```
期末净资产 ≈ 期初净资产
           + 鲨鱼收入                                  ← 经营性流入主体
           - 鲨鱼支出（剔除"债务还本"本金）            ← 经营性流出主体
           + cashflow_log 经营性流入                  ← 鲨鱼外的经营性收入
           - cashflow_log 经营性流出                  ← 鲨鱼外的经营性支出
           + Company_Stock NCF（SAP 归属薪酬）         ← 不经过银行账户的薪酬
           ± 资产估值变化（残差，应≈0）
           ─── 不含 Capital_Inflow/Outflow（资产形态转换不影响净资产）───
```

---

## 三、家庭三个数据源的角色

我们的家庭财务数据分散在三个独立系统里，每个系统记录**一部分**真实现金流。设计的关键是让三个系统**互补不重复**。

### 3.1 鲨鱼记账（`鲨鱼记账/<季度>.csv`）

**捕获范围**：所有经过你银行卡 / 支付宝 / 现金的交易。

| 记什么 | 不记什么 |
|--------|---------|
| 工资到手部分（含 RSU 税后现金部分）| ESPP 扣前截留的钱（直接被公司扣去买股票）|
| 房贷/车贷月供（"贷款"利息 + "债务还本"本金）| RSU 无偿归属的股票（无现金流）|
| 日常消费、孩子教育、医疗等 | 大额资产置换（如卖车）|
| 房租收入、退税 | 投资账户内部调仓（不出账）|

**关键约定（CLAUDE.md NCF 部分相关）**：
- 工资条目记录的是**到账后**的金额（ESPP 已被扣前截留）
- 6/19 那笔 ¥13,962 这类条目是 **RSU 归属时公司代扣税后剩余打到银行的现金**——本质是工资，不是 SAP 投资行为
- 备注里有 "sap股票" 的工资条目 = RSU 现金部分

### 3.2 cashflow_log.csv

**捕获范围**：鲨鱼记账漏掉的、家庭基金外的特殊现金流。

按 Type 字段分类：

| Type | 性质 | 例子 | 是否计入预测 |
|------|------|------|------------|
| `Inflow_Salary` | 经营性流入 | 季度奖、年终奖（暂用，未来自动化） | ✅ |
| `Inflow_Other` | 经营性流入 | 政府补贴、保险理赔、家人转账 | ✅ |
| `Outflow_Major` | 经营性流出 | 基金外大额支出（保险年费、装修首付等）| ✅ |
| `Capital_Inflow` | 资本性流入 | 卖车、卖房、资产变现 | ❌ |
| `Capital_Outflow` | 资本性流出 | 买车首付、大额资产购置 | ❌ |

**为什么资本性不计入预测**：

```
卖车 ¥207k 收到现金
  → 资产端: 车 -¥207k, 现金 +¥207k
  → 净资产变化 = 0 (一进一出)
```

balance_sheet.csv 已经反映了这次资产端的转移（车从清单移除，现金账户增加），所以预测公式**不应该再加一次** ¥207k——那是双重计算。

### 3.3 portfolio.csv（家庭基金）

**捕获范围**：投资账户内的所有持仓变化和现金流。

净资产核对**只用到一个特殊路径**：**SAP 归属薪酬**（`Asset_Class == 'Company_Stock'` 的 NCF）。

#### 为什么 SAP NCF 是关键

ESPP 和 RSU 归属的钱，**两个普通系统都看不到**：

| 真实流向 | 鲨鱼记账 | cashflow_log | portfolio.csv |
|---------|---------|--------------|---------------|
| ESPP（工资扣前截留 → 折扣价买股票）| ❌ | ❌ | ✅ NCF +Cost_CNY |
| RSU 无偿归属股票部分 | ❌ | ❌ | ✅ NCF +FMV |
| RSU 税后到手现金部分 | ✅ "工资" | ❌ | ❌ |

所以**必须从 portfolio.csv 提取 Q 内 Company_Stock NCF**，作为独立项加入预测，否则就会漏算这部分薪酬，让残差人为变大。

#### 剔除建仓日 NCF

建仓日（`Date == portfolio['Date'].min()`）的 NCF = 全部市值，是**成本基准点**，不是当期流入。必须剔除。

```python
mask = (
    (Asset_Class == 'Company_Stock')
    & (q_prev_end < Date <= q_curr_end)
    & (Date != inception_date)
)
sap_vesting = NCF[mask].sum()
```

## 四、设计演进（残差收敛的故事）

### V1（初版，有 bug）

把 cashflow_log 所有正数 Amount 当作 `special_inflow` 加入预测，无视 Type 字段。

```
Q2 预测 = 鲨鱼收入 - 鲨鱼支出(剔债务还本) + 所有正Amount - 所有负Amount
       = 202,487 - 129,623 + 222,000 - 0
       = 294,864
```

**问题**：把卖 X3 ¥207k 当作"净资产创造"，但实际是资产置换。

**Q2 残差：-¥191,877 (-1.27%)** ← 严重偏离

### V2（修复一：区分经营性 vs 资本性）

加 `_classify_cashflow_type()` 按 Type 字段分类：

```
Q2 预测 = 鲨鱼收入 - 鲨鱼支出(剔债务还本)
       + 经营性流入(¥15k 政府补贴)
       - 经营性流出(¥0)
       = 202,487 - 129,623 + 15,000 - 0
       = 87,864
```

**Q2 残差：+¥15,123 (+0.10%)** ← 大幅收敛，但还有漏算

### V3（修复二：补 SAP 归属薪酬）

新增 `portfolio_df` 参数，提取 Q 内 Company_Stock NCF：

```
Q2 SAP 归属（剔除建仓日 4/10）:
  5/8  ESPP    +¥3,292
  6/12 ESPP    +¥3,292
  6/19 RSU     +¥13,963
  ──────────
  合计         +¥20,547

Q2 预测 = 87,864(V2) + 20,547(SAP) = 108,411
```

**Q2 残差：-¥5,424 (+0.04%)** ← 最精确，捕捉到 SAP 股价跌+黄金跌等真实估值波动

## 六、易混淆点（给未来的自己看）

### Q：为什么不直接用 `工资到手` 推算 SAP NCF？

A：因为 RSU 现金部分 ≠ RSU NCF。鲨鱼里看到 ¥13,962 是**税后剩余打到银行**的部分；portfolio 里的 ¥13,963 是同一次 RSU 归属的**股票市值（FMV）**，按 CLAUDE.md 约定 NCF=FMV，作为成本基准。

这是巧合两个数字接近，本质是不同口径。不能用一个推算另一个。

### Q：ESPP NCF 是 Cost_CNY，会重复计算工资吗？

A：不会。ESPP 的 Cost_CNY 是**工资扣前截留**的部分，根本没进入鲨鱼记账的"工资"条目。所以鲨鱼"工资"+ portfolio "ESPP NCF" 是互补的，加起来才是完整薪酬。

### Q：为什么 "债务还本" 要从鲨鱼支出里剔除？

A：本金还款是**净资产转移**（现金 → 减少负债），不是消耗。balance_sheet.csv 的负债端已经反映了房贷/车贷余额的减少。如果不剔除，就会重复扣减一次。

利息部分（鲨鱼"贷款"分类）是**真实消耗**，正常算支出。

### Q：未来要补的 Q3+ 数据从哪里来？

A：每季末按 `docs/USER_MANUAL.md` 季度财报章节操作：

1. **balance_sheet.csv**：手动录入季末资产负债快照
2. **cashflow_log.csv**：录入鲨鱼外的特殊现金流（注意 Type 字段选对）
3. **鲨鱼记账/<季度>.csv**：从鲨鱼 APP 导出 UTF-16 CSV
4. **portfolio.csv**：Weekly Update 已自动捕捉 SAP 归属

整个净资产核对完全自动化，不需要手动填净资产变化解释。

## 八、约束和未来扩展

### 当前约束
- 桥接 portfolio.csv 时假设建仓日 = `Date.min()`。如果未来支持多账户或重启基线，需要重新定义"建仓日"的语义
- 未知 Type 按金额符号回退到经营性，是保守策略（避免漏算）。但若未来引入新 Type（如 `Inflow_Investment`），需在 `OPERATING_INFLOW_TYPES` 中注册

### 可能的未来扩展
- 残差自动拆解：用 portfolio.csv 的市值变化、balance_sheet 房产估值变化分别解释残差里的多少元
- 跨季度趋势线：净资产变化 = 经营性储蓄 / SAP 归属 / 资本性 / 估值变化 四组分趋势图
- 鲨鱼 vs cashflow_log 的重复校验：自动检测同一笔交易被双重记录
