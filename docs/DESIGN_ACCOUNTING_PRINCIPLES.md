# FamilyFund 家庭记账原则

> **版本**: v1.0  
> **日期**: 2026-04-27  
> **适用范围**: 鲨鱼记账 App + balance_sheet.csv + cashflow_log.csv 的协同使用规范

---

## 核心原则：资产负债表驱动

记账的目的不是记"花了多少钱"，而是记录**净资产的变化**。

每一笔交易都可以用以下框架分析：

```
△资产 = △负债 + △净资产（收益/损失）
```

**判断口诀：**

| 场景 | 本质 | 如何记录 |
|------|------|---------|
| 钱花出去，什么都没留下 | 净资产减少 | ✅ 鲨鱼记账：支出 |
| 钱出去，换回了资产 | 资产形态转换 | ❌ 不记支出，更新 balance_sheet 对应行 |
| 钱进来，来自外部（工资/补贴/租金） | 净资产增加 | ✅ 鲨鱼记账：收入 |
| 钱进来，来自借贷 | 负债增加，净资产≈不变 | ❌ 不记收入，更新 balance_sheet 负债行 |
| 负债还款（本金部分） | 负债减少，净资产≈不变 | ❌ 不记支出，更新 balance_sheet 贷款余额 |
| 负债还款（利息部分） | 净资产减少 | ✅ 鲨鱼记账：支出（贷款利息） |

---

## 三层数据分工

| 工具 | 记录内容 | 频率 | 用途 |
|------|---------|------|------|
| **鲨鱼记账** | 影响净资产的日常流量（收入/支出） | 每笔实时记录 | 季度消费汇总、Sankey 流图数据源 |
| **balance_sheet.csv** | 所有资产/负债的季末存量快照 | 每季末更新 | 家庭净资产计算、QoQ 对比 |
| **cashflow_log.csv** | 基金外特殊现金流（不经过鲨鱼记账的项目） | 按需，每季末检查 | 季度财报现金流补充 |

## 复杂案例：换车事件完整记账

> 场景：公司补贴购买私家车。前期支出约 ¥4万（首付+保险等），旧车卖出 ¥20.7万，
> 待收政府以旧换新补贴 ¥1.5万，新增贷款 ¥30万（月供 ¥5,000）。

### 逐笔拆解

**① 支付首付 ¥X万（假设首付 ¥3万，含在 ¥4万里）**

- 本质：现金 → 车辆（资产形态转换）
- 鲨鱼记账：❌ 不记
- balance_sheet：下季末 `Asset_RealEstate` 车辆行更新为新车估值；`Asset_Current` 现金行自动体现减少

**② 支付保险费 ¥1万（含在 ¥4万里）**

- 本质：消耗品，净资产减少
- 鲨鱼记账：✅ 记支出，类别「保险」，备注「新车保险」
- balance_sheet：无需操作（已反映在现金余额减少）

**③ 旧车卖出 ¥20.7万到账**

- 本质：资产变现。若旧车在 balance_sheet 里有估值（如 ¥15万），差额 ¥5.7万是收益
- 鲨鱼记账：✅ 记收入，类别「其他收入」，金额 ¥20.7万，备注「旧车出售」
- balance_sheet：下季末 `Asset_RealEstate` 旧车行删除或清零

> **注**：若旧车估值和实际卖出价有差异，鲨鱼记账的收入金额以**实际到账**为准，
> 账面差异在 balance_sheet 里通过删除旧车行自然消除。

**④ 贷款 ¥30万放款**

- 本质：负债增加，现金增加，净资产≈不变（略有手续费损耗）
- 鲨鱼记账：❌ 不记
- balance_sheet：`Liability_LongTerm` 新增贷款行，金额 ¥30万；`Asset_Current` 现金短暂增加（随即支付车款）

**⑤ 每月还款 ¥5,000**

- 本质：本金部分（约 ¥4,500）= 负债减少；利息部分（约 ¥500）= 净资产减少
- 鲨鱼记账：✅ 仅记利息部分，类别「贷款利息」，备注「车贷利息」
- balance_sheet：每季末查银行 App 更新 `Liability_LongTerm` 贷款剩余本金（季末余额直接填，不用手算）

> **简化处理**：如果本金/利息拆分麻烦，可以整笔 ¥5,000 记为支出「贷款还款」，
> 理解为"这笔钱从可支配收入里消失了"，balance_sheet 里的负债余额靠季末查银行更新。
> 两种口径都可以，选一种保持一致即可。

**⑥ 政府以旧换新补贴 ¥1.5万到账**

- 本质：外部转入，净资产增加
- 鲨鱼记账：✅ 记收入，类别「其他收入」，备注「以旧换新政府补贴」
- balance_sheet：无需单独操作（已反映在现金余额）

### 换车事件汇总

| 项目 | 鲨鱼记账 | balance_sheet 变化 |
|------|---------|-------------------|
| 首付 ¥3万 | ❌ 不记 | 车辆资产+，现金- |
| 保险 ¥1万 | ✅ 支出 ¥1万 | 无 |
| 旧车卖出 ¥20.7万 | ✅ 收入 ¥20.7万 | 旧车资产行清零 |
| 贷款 ¥30万 | ❌ 不记 | 长期负债+ ¥30万 |
| 月供利息 ≈¥500/月 | ✅ 支出（利息部分） | 每季末更新贷款余额 |
| 政府补贴 ¥1.5万 | ✅ 收入 ¥1.5万 | 无 |

## 工资与收入的记录口径

### 记录原则

鲨鱼记账的收入 = **税后实际到账金额**，不区分谁的工资，统一记入家庭账本。

| 收入类型 | 鲨鱼记账 | 备注 |
|---------|---------|------|
| 工资（税后到账） | ✅ 记，类别「工资」 | 含奖金，以实际到账为准 |
| SAP ESPP/RSU 归属 | ✅ 如有现金到账则记 | 股票形式归属不记（已在 portfolio.csv 追踪） |
| 公积金（单位部分） | ❌ 不记 | 不经过你的银行账户，季末查余额更新 balance_sheet |
| 公积金（个人部分扣除） | ❌ 不记 | 已在税前扣除，工资到账金额已是税后 |
| 公积金（实际提取到账） | ✅ 记收入 | 类别「公积金」 |
| 房租收入 | ✅ 记收入 | 类别「房租」 |
| 理财/基金分红现金到账 | ✅ 记收入 | 类别「投资收益」 |
| 家庭基金取出 | ❌ 不记 | 已在 portfolio.csv Cash NCF 里追踪 |

## cashflow_log.csv 的定位

`cashflow_log.csv` **只补录鲨鱼记账捕捉不到的特殊项**，大多数季度可以为空。

需要填的情况：
- 大额现金交易（没有走银行，鲨鱼记账没有记录）
- 跨境转账
- 家庭内部大额借款/还款（如对父母的借款）

**不需要填的情况：**
- 工资、房租、消费 → 已在鲨鱼记账
- 家庭基金注资/取出 → 已在 portfolio.csv Cash NCF
- 贷款本金变化 → 体现在 balance_sheet 负债余额

## 季末对账流程（约 15-20 分钟）

1. **鲨鱼记账导出**：导出当季明细 CSV，放入 `$FAMILYFUND_DATA/<季度>/鲨鱼记账明细.csv`
2. **更新 balance_sheet.csv**：查各账户余额，更新存量快照（详见 USER_MANUAL.md 第 6 节）
3. **检查 cashflow_log.csv**：有无遗漏的基金外特殊项
4. **核对公式**：
   ```
   期末净资产 ≈ 期初净资产 + 鲨鱼收入合计 - 鲨鱼支出合计 ± 资产估值变化
   ```
   误差在 ¥5,000 以内属正常（估值误差、未记小额等）

## Sankey 家庭资金流图（未来功能）

当 cashflow_log 和鲨鱼记账数据积累 2-3 个季度后，可以做：

```
[流入]                    [流出/沉淀]
工资收入       ──────────→ 餐饮消费
公积金提取     ──────────→ 交通
房租收入       ──────────→ 孩子相关
其他收入       ──────────→ 购物
               ──────────→ 家庭基金注资
               ──────────→ 净储蓄增加
```

**前置条件**：
- 鲨鱼记账完整记录（已满足）
- 工资口径统一（已明确）
- balance_sheet 至少 2 个季度（已有 25Q4 + 26Q1）
