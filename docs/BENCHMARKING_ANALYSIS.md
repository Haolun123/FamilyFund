# Benchmarking Analysis for FamilyFund

> 评估基准指数与宏观经济指标在家庭基金中的适用性

## 1. Market Benchmarks（市场基准）

### 1.1 CSI 300（沪深300，000300.SH）

| 维度 | 说明 |
|------|------|
| **追踪标的** | 沪深两市市值最大、流动性最好的 300 只 A 股，自由流通市值加权 |
| **数据源** | `akshare`: `ak.stock_zh_index_daily(symbol="sh000300")`；`yfinance`: `000300.SS`；Tushare (free tier) |
| **频率** | 每交易日收盘（15:00 CST） |
| **货币** | CNY，无需汇率换算 |
| **与本基金的关联性** | ⭐⭐⭐ 直接相关 — 基金持有 A500 场外、红利低波 ETF、成都银行等 A 股资产 |

**注意事项：**
- 默认是价格指数（不含分红），完全对标需使用全收益指数 H00300.CSI（免费源较少）
- A 股有春节、国庆等长假停市，对比时注意日期对齐
- 作为蓝筹基准，无法覆盖小盘股表现；若需全市场对比，参见万得全A

---

### 1.2 S&P 500（标普500，^GSPC）

| 维度 | 说明 |
|------|------|
| **追踪标的** | 美国 500 家大型上市公司，覆盖约 80% 美股市值 |
| **数据源** | `yfinance`: `^GSPC`；Alpha Vantage (free, 25 req/day)；FRED: `SP500`；`akshare` |
| **频率** | 每交易日收盘（16:00 EST） |
| **货币** | USD — **必须转换为 CNY 才能与基金净值对比** |
| **与本基金的关联性** | ⭐⭐⭐ 直接相关 — 基金持有大量标普/纳指场外基金 |

**注意事项：**
- **时区错位**：中国交易日收盘（15:00 CST）时美股尚未开盘（02:00 EST），相当于对比"今天的基金 vs 昨天的标普"。周/月频率下可接受，日频有噪声。
- **CNY 转换**：`S&P500_CNY(t) = S&P500(t) × USDCNY(t)`，然后归一化到基金起始日=1.0。汇率源推荐 PBOC 中间价（`akshare` 提供）。
- 标普上涨 10% USD ≠ 10% CNY — 汇率波动可能放大或抵消收益。这恰好是对比的意义所在：衡量"以人民币计价买美股到底赚了多少"。

---

### 1.3 Wind All-A（万得全A，881001.WI）

| 维度 | 说明 |
|------|------|
| **追踪标的** | 沪深全部 A 股（5000+ 只），自由流通市值加权，是中国最广谱的股票基准 |
| **数据源** | `akshare`: `ak.stock_zh_index_daily(symbol="sh881001")`（需验证可用性）；Tushare Pro（付费）；Wind 终端（商业） |
| **频率** | 每交易日 |
| **货币** | CNY |
| **与本基金的关联性** | ⭐⭐ 中等 — 比 CSI300 更全面，但基金 A 股持仓偏蓝筹，全A可能不是最精确基准 |

**注意事项：**
- **数据获取是最大风险**。万得全A 是商业指数，`akshare` 免费渠道可能不稳定或被封。
- **备选方案**：中证全指（000985.CSI），由中证公司公开发布，覆盖范围与万得全A接近，数据源更可靠。
- 建议：主选万得全A，代码层面预留中证全指作为 fallback。

## 2. Economic Indicators（宏观经济指标）

### 2.1 CPI — 居民消费价格指数

| 维度 | 说明 |
|------|------|
| **发布频率** | 月度，次月 9-12 日由国家统计局（NBS）发布 |
| **数据格式** | NBS 发布同比 (%)、环比 (%)；FRED 提供累计指数 `CHNCPIALLMINMEI`（2015=100） |
| **数据源** | FRED（推荐，直接是指数）；`akshare`: `ak.macro_china_cpi()`；NBS data.stats.gov.cn |
| **与本基金的关联性** | ⭐⭐⭐ 高度相关 — 回答"基金跑赢通胀了吗？"这个核心问题 |

**如何与 NAV 对比：**
1. 获取 FRED 的 CPI 指数序列（已是累计值，2015年=100）
2. 在基金起始日归一化为 1.0：`CPI_normalized(t) = CPI(t) / CPI(start_date)`
3. 与基金 NAV 绘制在同一张图上
4. CPI 是月度数据 — 在 NAV 图上按月显示节点即可，不必日频插值

**实际价值**：
- 假设基金净值增长 15%，CPI 同期累计增长 5%，则实际购买力增长约 10%
- 这是家庭基金最重要的基准之一 — 跑不赢通胀等于亏钱
- 尤其对固定收益占比 50%+ 的本基金，CPI 基准非常有意义

---

### 2.2 M2 — 广义货币供应量

| 维度 | 说明 |
|------|------|
| **发布频率** | 月度，次月 10-15 日由中国人民银行（PBOC）发布 |
| **数据格式** | PBOC 发布 M2 余额（万亿元）和同比增速 (%)；`akshare` 提供历史序列 |
| **数据源** | `akshare`: `ak.macro_china_money_supply()`；PBOC 官网；FRED `MYAGM2CNM189N` |
| **与本基金的关联性** | ⭐⭐⭐ 高度相关 — 回答"基金跑赢印钞速度了吗？"这个更高标准的问题 |

**为什么 M2 比 CPI 更严格：**
- CPI 衡量消费品价格（食品、服装、交通），反映"购买力保值"
- M2 衡量整个经济体的货币扩张速度，反映"相对财富稀释"
- 2020-2025 年，中国 M2 同比增速 8-12%，而 CPI 仅 0-2%——差值流入了房产、股市、理财产品等资产价格
- 一个跑赢 CPI 但跑不过 M2 的基金：你的"购物能力"没降，但你在社会财富中的"占比"缩水了

**如何与 NAV 对比：**
1. 获取 M2 余额月度序列（akshare 提供绝对值，单位万亿元）
2. 在基金起始日归一化为 1.0：`M2_normalized(t) = M2(t) / M2(start_date)`
3. 与基金 NAV 绘制在同一张图上
4. 或者：用 M2 YoY% 链式计算累计指数 — `Index(t) = Index(t-1) × (1 + YoY(t)/100/12)`（月度化）

**实际价值：**
- 对本基金尤其重要：50%+ 固定收益配置，年化收益约 3-4%，而 M2 增速 8-10%
- 如果长期跑不赢 M2，说明配置过于保守，需要提高权益占比
- M2 是家庭基金的"更高标准线"：CPI 是及格线，M2 是优秀线

---

### 2.4 PPI — 工业品出厂价格指数

| 维度 | 说明 |
|------|------|
| **发布频率** | 月度，与 CPI 同步发布 |
| **数据源** | NBS；FRED `PITGCG01CNM661N`；`akshare`: `ak.macro_china_ppi()` |
| **与本基金的关联性** | ⭐ 低 — 衡量生产端成本，不反映家庭购买力 |

**评估：**
- PPI 用于判断工业周期和上下游传导，对宏观研究有价值
- 但对家庭基金的核心问题（"我的钱有没有增值"）没有直接回答
- PPI 与 CPI 走势经常分化（如 2022-2023 年 PPI 通缩、CPI 温和），单独看 PPI 可能误导

**结论：不建议纳入基准体系。**

---

### 2.5 GDP Deflator — GDP 平减指数

| 维度 | 说明 |
|------|------|
| **发布频率** | 季度，滞后 1-2 个月，且有修正 |
| **数据源** | NBS 不直接发布，需用 `名义GDP / 实际GDP × 100` 计算；世界银行提供年度数据 |
| **与本基金的关联性** | ⭐ 低 — 理论上最全面的物价指标，但实操性差 |

**评估：**
- 理论优势：覆盖全经济体价格变动（不只消费品），是"最广义的通胀"
- 实操劣势：
  - 季度频率 — 基金 NAV 至少周频更新，对齐困难
  - 发布滞后 — Q1 数据要到 5-6 月才确认
  - 需手动计算 — 无免费直接指数源
  - 容易被 GDP 统计口径调整影响
- CPI 已经能回答"跑赢通胀"的问题，GDP 平减指数的额外信息量不值得其实现成本

**结论：不建议纳入。CPI 是更好的替代。**

## 3. Summary & Recommendation

### 推荐纳入（按优先级排序）

| 优先级 | 基准 | 类型 | 用途 | 数据源 |
|--------|------|------|------|--------|
| P0 | **CPI 累计指数** | 宏观 | 通胀基准线 — 基金的最低合格线（"跑赢物价了吗？"） | FRED `CHNCPIALLMINMEI` |
| P0 | **M2 货币供应量** | 宏观 | 货币扩张基准线 — 基金的更高标准（"跑赢印钞了吗？"） | `akshare` / FRED |
| P0 | **CSI 300** | 市场 | A 股基准，对标国内权益持仓 | `akshare` / `yfinance` |
| P0 | **S&P 500 (CNY)** | 市场 | 美股基准，对标标普/纳指基金 | `yfinance` + PBOC FX |
| P1 | **Wind All-A** | 市场 | 全 A 股基准，比 CSI300 更广 | `akshare`（不稳定，需 fallback 中证全指） |

### 不建议纳入

| 指标 | 原因 |
|------|------|
| PPI | 衡量生产端成本，对家庭购买力无直接参考价值 |
| GDP 平减指数 | 季度发布、滞后严重、需手动计算，CPI 已足够替代 |

### Implementation Notes

**归一化方法：** 所有基准在基金起始日归一化为 1.0，与 NAV 起始值对齐：
```
Benchmark_normalized(t) = Benchmark(t) / Benchmark(fund_start_date)
```

**S&P 500 CNY 转换：**
```
SP500_CNY(t) = SP500_USD(t) × USDCNY(t)
SP500_normalized(t) = SP500_CNY(t) / SP500_CNY(fund_start_date)
```

**CPI 处理：** 月度数据，建议按月末节点标注在 NAV 图上，不做日频插值（保持数据诚实）。

**M2 处理：** 月度数据，使用 M2 余额绝对值归一化（与 CPI 处理方式一致）。M2 增速远快于 CPI，图上会明显分开 — 这正是对比的价值所在。

**展示建议：** Dashboard 新增 "Benchmark" tab 或在 Fund NAV tab 上叠加基准曲线。用户可勾选显示/隐藏各基准线。建议默认显示 CPI 和 M2 两条宏观线 + 一条市场基准线（CSI300 或 S&P500 取决于用户配置）。
