# DESIGN: 弹药池与现金流压力测试（Ammo Pool）

> **状态**：待实现（P2）
> **日期**：2026-05-12

---

## 功能定位

矩阵策略的隐含假设是"极端加仓时有足够现金"。本工具回答一个问题：**按当前配置和信号，我的弹药还能撑多久？**

不修改策略逻辑，只做信息展示和预警。

---

## 核心概念

### 弹药池

```
弹药池 = Cash 余额 + Fixed_Income 可赎回市值 - 紧急备用金
```

| 组成 | 来源 | 说明 |
|------|------|------|
| Cash 余额 | `portfolio.csv` Cash 行 `Total_Value` | 随时可用 |
| Fixed_Income | `portfolio.csv` Fixed_Income 行 `Total_Value` 之和 | T+2 可用（短债/理财赎回），极端行情时两天不影响 |
| 紧急备用金 | 配置项，默认 ¥200,000 | 刚性扣除，代表不可动用的生活保障金 |

**紧急备用金逻辑**：即使所有信号同时顶格，这笔钱绝对不动。代表约3-6个月家庭基本支出。默认20万，可在 UI 调整。

### 不约束单标的占比

矩阵策略的核心价值是逆向加仓。如果在最好的买点因为"占比够了"而不买，策略就丧失意义。占比会在市场恢复时自然回归（跌得多的涨得多）。

**约束放在弹药总量上，而不是单标的占比上。**

## 功能一：弹药健康度实时展示

### 位置

Market Tab → DCA Plan 汇总表下方（"本周建议总投入"旁边）

### 展示内容

```
─── 弹药健康度 ─────────────────────────────────
弹药池：¥182,000（Cash ¥98,000 + 固收 ¥304,000 - 备用金 ¥200,000）
本周消耗：¥3,100（按当前矩阵信号）
月消耗速率：¥12,400（按当前信号持续）

可支撑极端加仓：14.7 周（按当前信号）
                8.2 周（按全部顶格）

[⚠️ 弹药不足时自动显示警告]
```

### 信号颜色

| 弹药池状态 | 颜色 | 说明 |
|-----------|------|------|
| > 全部顶格可撑8周 | 🟢 绿色 | 健康，可全力执行 |
| 4-8周 | 🟡 黄色 | 关注，考虑是否需要补充 |
| < 4周 | 🔴 红色 | 警告，建议降档或补充现金 |

> 8周基准来源：2008年金融危机 VIX>30 持续约8周，是历史最极端连续加仓期。

## 功能二：压力测试计算器

### 位置

DCA Plan 区域内，折叠区"⚡ 压力测试"

### 输入

| 参数 | 来源 | 默认值 |
|------|------|--------|
| Cash 余额 | 自动读 portfolio.csv | — |
| Fixed_Income 市值 | 自动读 portfolio.csv | — |
| 紧急备用金 | 用户配置 | ¥200,000 |
| 各标的基准金额 | DCA Manager 自动读 | — |
| 极端情景持续周数 | 用户输入 | 8 周 |
| 极端信号强度 | 用户选择 | 全部顶格（5个标的同时 10x/5x） |
| 月新增储蓄 | 读 fi_config.json | — |

### 输出

```
─── 压力测试：全部顶格持续 8 周 ──────────────────
每周消耗：¥33,500（标普10x + 纳指10x + 沪深10x + A500·10x + 黄金5x）
8周总消耗：¥268,000

弹药池起始：¥182,000
月新增储蓄补充：+¥24,000（2个月）
8周后弹药池：¥-62,000 ← ⚠️ 不足！

临界周数：第 5.4 周弹药耗尽
建议：
  - 方案A：增加紧急备用金至 ¥150,000（释放 ¥50,000 弹药）→ 可撑7.0周
  - 方案B：基准金额全部降50% → 16周无压力
  - 方案C：保持现状，接受第6周后被迫暂停风险
```

### 可视化

弹药池逐周递减折线图：
- X轴：周数
- Y轴：弹药池余额
- 红色虚线：0 线（耗尽线）
- 绿色虚线补充：月储蓄的阶梯式注入

## 功能三：动态降档建议（仅建议，不自动执行）

当弹药池低于"全部顶格可撑4周"时，展示降档建议：

```
⚠️ 弹药不足警告
当前弹药可支撑全部顶格 3.2 周，建议降档：
  建议有效倍数 = 矩阵倍数 × 72%（= 弹药池 / 4周消耗量）
  
  博时标普500：原始 1.5x → 建议 1.1x
  南方纳指100：原始 1.5x → 建议 1.1x
  ...
```

**不自动修改任何配置**，仅展示供参考。执行决策权在用户。

## 设计决策

- **不约束单标的占比**：矩阵策略的核心是逆向加仓，占比约束会在最好的时刻阻止执行。约束放在弹药总量上。
- **不自动降档**：降档是重大策略变更，需要人工判断。工具只提供信息和建议。
- **固收视为可用弹药**：T+2 赎回在大规模回撤场景（通常持续数周）下不构成约束。
- **8周作为极端基准**：2008年金融危机是有记录以来最长的极端连续加仓期。
