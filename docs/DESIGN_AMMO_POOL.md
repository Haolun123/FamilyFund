# DESIGN: 弹药池与现金流压力测试（Ammo Pool）

> **状态**：待实现（P2）
> **日期**：2026-05-12

---

## 功能定位

矩阵策略的隐含假设是"极端加仓时有足够现金"。本工具回答一个问题：**按当前配置和信号，我的弹药还能撑多久？**

不修改策略逻辑，只做信息展示和预警。

---

## 核心概念

### 弹药池

```
弹药池 = Cash 余额 + Fixed_Income 可赎回市值 - 紧急备用金
```

| 组成 | 来源 | 说明 |
|------|------|------|
| Cash 余额 | `portfolio.csv` Cash 行 `Total_Value` | 随时可用 |
| Fixed_Income | `portfolio.csv` Fixed_Income 行 `Total_Value` 之和 | T+2 可用（短债/理财赎回），极端行情时两天不影响 |
| 紧急备用金 | 配置项，默认 ¥200,000 | 刚性扣除，代表不可动用的生活保障金 |

**紧急备用金逻辑**：即使所有信号同时顶格，这笔钱绝对不动。代表约3-6个月家庭基本支出。默认20万，可在 UI 调整。

### 不约束单标的占比

矩阵策略的核心价值是逆向加仓。如果在最好的买点因为"占比够了"而不买，策略就丧失意义。占比会在市场恢复时自然回归（跌得多的涨得多）。

**约束放在弹药总量上，而不是单标的占比上。**

## 设计决策

- **不约束单标的占比**：矩阵策略的核心是逆向加仓，占比约束会在最好的时刻阻止执行。约束放在弹药总量和权益总敞口上。
- **不自动降档**：降档是重大策略变更，需要人工判断。工具只提供信息和建议。
- **固收视为可用弹药**：T+2 赎回在大规模回撤场景（通常持续数周）下不构成约束。
- **8周作为极端基准**：2008年金融危机是有记录以来最长的极端连续加仓期。

---

## 权益总敞口约束（集中度风险控制）

### 为什么需要

矩阵策略"不约束单标的占比"只是说加仓时不因占比而暂停。但各标的之间有**相关性**——美股+A股在系统性危机中一起跌（2008年纳指-50%、标普-57%、沪深300-72%同时发生），单标的分散不等于风险分散。

真正需要约束的是**权益类总敞口**：美股+A股+个股合计占比。

### 风险容忍反推

用户风险容忍：**组合最大可承受回撤 -30%**

极端情景假设（2008级，历史最极端）：

| 类别 | 跌幅假设 |
|------|---------|
| 权益类（美股+A股+个股）| -60% |
| 黄金 | -10%（08年先跌后涨）|
| 固收+现金 | -2% |

反算：
```
-30% = (-60%) × E + (-10%) × 0.10 + (-2%) × (0.90 - E)
-30% = -58%E - 2.8%
E = 46.9% ≈ 47%
```

**权益类总敞口上限：~50%**（取整）

交叉验证（2022级中等危机：权益-30%，黄金-20%，固收-3%）：
```
组合回撤 = -30%×0.47 + (-20%)×0.10 + (-3%)×0.43 = -17.4%
```
中等危机下只跌-17%，远未到-30%容忍线，安全余量充足。

### 实操规则

| 权益总占比 | 状态 | 动作 |
|-----------|------|------|
| < 40% | 🟢 安全 | 矩阵信号正常执行 |
| 40% - 50% | 🟡 关注 | 矩阵信号正常执行，但提示"接近上限" |
| ≥ 50% | 🔴 超限 | 权益类矩阵信号暂停，不强制卖出 |

> "暂停"不是"卖出"——只是不再加仓，等待市场自然回调或其他资产增长带来的占比回归。

### 与再平衡建议的联动

当前再平衡建议基于"目标配置比例"做偏差计算。新增权益总敞口约束后，需要加一层红线检查：

```
再平衡建议流程：
  1. 按目标配置比例计算各类别偏差
  2. 检查权益总占比：
     if 当前权益总占比 >= 50%:
       权益类不再显示"建议买入"
       改为显示"⚠️ 权益总敞口已达上限，暂停加仓"
     else:
       正常显示建议
```

同时，用户设定目标配置时：
- 如果权益类合计目标 > 50%，显示提示"权益目标占比超出风险容忍（基于-30%回撤容忍推算），建议调低"
- 不强制阻止（用户可能有自己的判断），但确保知情
