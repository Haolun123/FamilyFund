# DESIGN: 人生阶段规划（Life Stage Planning）

> **状态**：待实现（P3）
> **日期**：2026-05-02

---

## 功能定位

不是预测，是**情景规划**（Scenario Planning）。

人生里程碑支出高度不确定，但可以设定悲观/基准/乐观三种情景，观察不同情景下财务独立时间线的漂移幅度。输出直接对接 `DESIGN_ANALYTICS.md` 中的财务独立测算模块，替代其静态的 `annual_expense_target_cny` 字段。

---

## 核心设计原则

- **情景而非预测**：每个里程碑设三档（悲观/基准/乐观），不给单一确定数字
- **动态支出曲线**：按年份生成支出序列，替代静态年支出
- **参考指标锚定**：大额支出用可观测的公开指标做锚（房价/CPI/留学成本），减少拍脑袋
- **与投资系统解耦**：里程碑规划是独立配置，不影响 portfolio.csv 的任何计算

---

## 人生里程碑框架

| 阶段 | 默认时间窗口 | 主要支出项 | 参考指标 |
|------|------------|-----------|---------|
| **早期养育** | 孩子 0-6 岁 | 奶粉、早教、医疗、托育 | CPI（居民消费）|
| **基础教育** | 孩子 6-18 岁 | 学费、补习、兴趣班、夏令营 | 教育类 CPI |
| **高等教育** | 孩子 18-22 岁 | 国内大学 or 出国留学 | 留学目的地生活成本 |
| **置业** | 用户自定义 | 首付、月供、装修、物业 | 一二线城市房价、LPR |
| **购车** | 用户自定义 | 购置税、保险、维护 | — |
| **父母养老** | 用户自定义 | 医疗、护理、辅助生活 | 医疗类 CPI |
| **自身退休** | 60 岁+ | 日常生活、旅行、医疗 | CPI、M2 |

每个里程碑可独立启用/禁用，时间窗口可调整。

---

## 情景设定

每个里程碑设三档：

| 档位 | 含义 | 示例（高等教育） |
|------|------|---------------|
| 🔴 悲观 | 最贵路径 | 美国/英国留学，4年 ¥250万 |
| 🟡 基准 | 最可能路径 | 国内985 + 海外1年交换，¥30万 |
| 🟢 乐观 | 最省路径 | 国内普通本科 + 奖学金，¥15万 |

用户对每个里程碑选择一个档位，系统自动生成对应年份的支出分摊。

## 参考指标体系

### 房产置业
- **一线城市均价**：北上广深，参考链家/贝壳数据（手动录入，每年更新一次）
- **二线城市均价**：成都/杭州/武汉等
- **LPR（贷款市场报价利率）**：影响月供计算，可从人民银行官网拉取或手动录入
- **首付比例**：当前政策（首套/二套）

### 通胀调整
- **CPI**：国家统计局，用于将今天的估算值折算到未来年份
- **M2 增速**：货币超发对资产价格的影响，主要用于房产估值参考
- **教育通胀**：历史上教育成本涨速高于 CPI，单独设定（默认 +2% over CPI）
- **医疗通胀**：同上，默认 +3% over CPI

### 留学成本参考
| 目的地 | 年均总费用（学费+生活）| 备注 |
|--------|-------------------|------|
| 美国（私立）| $80,000-$100,000 | |
| 英国 | £35,000-£50,000 | |
| 澳大利亚 | AUD $50,000-$70,000 | |
| 新加坡 | SGD $40,000-$60,000 | 热门选项 |
| 香港 | HKD $250,000-$350,000 | |
| 国内985 | ¥5,000-$10,000/年 | |

以上为参考基准，用户可覆盖。

## 输出：动态支出曲线

系统将 `life_stages.json` 展开为逐年支出序列：

```
2026: 基础生活 + 早期养育（基准）= ¥26万
2027: 基础生活 + 早期养育（基准）= ¥26万
...
2030: 基础生活 + K12教育 + 置业首付（一次性）= ¥200万
2031: 基础生活 + K12教育 + 月供 = ¥34万
...
2042: 基础生活 + 高等教育 = ¥35万
...
2055: 退休生活支出（基准）= ¥18万
```

这条曲线替代财务独立测算里的静态 `annual_expense_target_cny`，使 FI 测算结果更真实。

## 实现顺序

1. 先手动维护 `life_stages.json`（用户直接编辑），不做 UI
2. 财务独立测算做好后，接入动态支出曲线
3. 最后做 UI 编辑器（情景选择、参数调整）

这样可以先用数据感受结果，再投入 UI 开发成本。
